Initial commit: PagerDuty Datacenter BMS Integration

Complete Terraform configuration for PagerDuty datacenter demo with Niagara BMS integration.

Features:
- 6 services (Power, HVAC, Environmental, Security, Fire, Network)
- 4 response teams with 15 demo users
- 7 on-call schedules (Australia/Sydney timezone)
- 6 escalation policies with dual notification (phone + SMS)
- Advanced event orchestrations with intelligent routing
- Water leak detection with immediate escalation
- Maintenance window suppression
- Critical equipment pattern matching (UPS, CRAC, Chiller, Core Network)
- MS Teams integration for incident notifications
- Intelligent alert grouping to reduce fatigue

Configuration includes:
- main.tf - Provider and data sources
- variables.tf - Variable definitions
- terraform.tfvars - Customizable values
- users.tf - 15 demo users with contact methods
- teams.tf - 4 response teams
- schedules.tf - 24/7 on-call coverage
- escalation_policies.tf - Tiered response with dual notification
- services.tf - 6 services with alert grouping
- event_orchestrations.tf - Advanced alert routing and enrichment
- outputs.tf - Integration keys and service IDs
- .gitignore - Protects sensitive files
- README.md - Complete documentation

Timezone: Australia/Sydney
Notification: MS Teams + Phone/SMS for critical alerts
Alert Suppression: Maintenance mode support across all services

Ready for terraform init && terraform apply
